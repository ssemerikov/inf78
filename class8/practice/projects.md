# Проекти — Цифрові пристрої

---

## Проєкт 1. Збирання ПК

**Мета:** навчитися підбирати компоненти комп'ютера залежно від його призначення та обґрунтовувати свій вибір на основі технічних характеристик.

### Завдання

Оберіть один із сценаріїв та підберіть конфігурацію комп'ютера:

- **Сценарій А — Офісний комп'ютер:** для роботи з документами, інтернетом, онлайн-навчання.
- **Сценарій Б — Ігровий комп'ютер:** для сучасних ігор та потокового відео.
- **Сценарій В — Серверна робоча станція:** для 3D-моделювання, відеомонтажу та розробки ПЗ.

### Етапи виконання

1. Визначте вимоги до комп'ютера для обраного сценарію (які задачі він вирішуватиме, які програми працюватимуть).
2. Підберіть усі необхідні компоненти: процесор, оперативну пам'ять, накопичувач, відеокарту, материнську плату, блок живлення, монітор.
3. Перевірте сумісність компонентів: тип сокета, стандарт RAM, потужність блоку живлення (з запасом 20–30%).
4. Складіть таблицю конфігурації з технічними характеристиками кожного компонента та орієнтовною вартістю.
5. Напишіть обґрунтування: чому саме ці компоненти підходять для обраного сценарію.

### Очікуваний результат

Таблиця конфігурації комп'ютера (компонент, модель, характеристики, вартість) та письмове обґрунтування вибору (5–8 речень). Загальна сума конфігурації не повинна перевищувати реалістичний бюджет для обраного сценарію.

## Проєкт 2. Діагностика комп'ютера

**Мета:** навчитися використовувати вбудовані засоби діагностики Windows для аналізу стану апаратного та програмного забезпечення комп'ютера.

### Завдання

Виконайте діагностику вашого комп'ютера (домашнього або шкільного), зафіксуйте результати та зробіть висновки.

### Етапи виконання

1. Відкрийте **Диспетчер задач** (Ctrl+Shift+Esc). Зафіксуйте завантаження CPU, RAM, диска та мережі у стані спокою. Запустіть браузер із кількома вкладками та зафіксуйте зміни.
2. Відкрийте **Диспетчер пристроїв**. Перевірте наявність пристроїв із попередженнями (жовтий трикутник). Зафіксуйте список усіх основних категорій пристроїв.
3. Запустіть **Монітор ресурсів**. Проаналізуйте, які процеси споживають найбільше ресурсів CPU та пам'яті.
4. Відкрийте інформацію про систему (Win+Pause). Зафіксуйте: версію ОС, процесор, об'єм RAM, тип системи (32/64 біт).
5. За можливості перевірте температуру процесора (через BIOS/UEFI або безкоштовну програму HWMonitor). Зафіксуйте температуру в стані спокою та під навантаженням.
6. Запустіть **Перевірку пам'яті Windows** (Windows Memory Diagnostic). Зафіксуйте результат.

### Очікуваний результат

Звіт у вигляді таблиці з результатами кожного етапу діагностики. Висновки (3–5 речень): загальний стан комп'ютера, наявність проблем, рекомендації щодо покращення (наприклад, додати RAM, замінити HDD на SSD, оновити драйвери).

## Проєкт 3. Розумний дім

**Мета:** спроєктувати систему Інтернету речей (IoT) для житлового будинку, описати пристрої, протоколи зв'язку та потік даних.

### Завдання

Спроєктуйте IoT-систему для квартири або будинку. Виберіть конкретний сценарій (наприклад, квартира-студія студента, будинок сім'ї з дітьми, дачний будинок).

### Етапи виконання

1. Визначте зону проєктування (кількість кімнат, призначення) та перелік задач, які має вирішувати система (освітлення, опалення, безпека, побутові прилади).
2. Підберіть IoT-пристрої для кожної задачі: датчики (температури, руху, вологості, відкриття дверей), виконавчі механізми (розумні вимикачі, термостати, замки), хаб/шлюз.
3. Оберіть протоколи зв'язку для кожного пристрою: Wi-Fi (висока швидкість), Zigbee/Z-Wave (mesh-мережа, мале споживання), BLE (для носимих пристроїв). Обґрунтуйте вибір.
4. Опишіть потік даних для однієї автоматизованої дії. Наприклад: «Датчик руху виявляє рух → передає сигнал через Zigbee на хаб → хаб надсилає команду через Wi-Fi до хмарного сервісу → сервіс повертає команду вмикнути світло → розумна лампочка вмикається».
5. Складіть схему системи: пристрої, їх розташування, зв'язки між ними.
6. Визначте заходи безпеки: паролі, шифрування, оновлення ПЗ, сегментація мережі.

### Очікуваний результат

Схема IoT-системи (можна намалювати від руки та сфотографувати) з позначеними пристроями та зв'язками. Таблиця пристроїв (пристрій, протокол, зона, функція). Опис одного автоматизованого сценарію з повним потоком даних (5–7 кроків). Перелік заходів безпеки (3–5 пунктів).

## Проєкт 4. ШІ у моєму житті

**Мета:** дослідити застосування штучного інтелекту в обраній галузі та проаналізувати переваги та ризики.

### Завдання

Оберіть галузь, яка вас цікавить, і дослідіть, як ШІ використовується у цій галузі.

Можливі галузі: освіта, медицина, транспорт, фінанси, мистецтво, сільське господарство, екологія, спорт.

### Етапи виконання

1. Оберіть галузь та сформулюйте питання для дослідження (наприклад: «Як ШІ допомагає лікарям ставити діагноз?» або «Як ШІ змінює музичну індустрію?»).
2. Знайдіть 3–5 реальних прикладів використання ШІ у цій галузі. Використовуйте надійні джерела (статті, офіційні сайти компаній, наукові публікації).
3. Для кожного прикладу опишіть: що робить ШІ, які дані використовує для навчання, який результат отримують користувачі.
4. Проаналізуйте переваги (щонайменше 3) та ризики (щонайменше 3) використання ШІ у цій галузі.
5. Сформулюйте власну позицію: чи варто використовувати ШІ у цій галузі, які обмеження необхідні. Обґрунтуйте.

### Очікуваний результат

Дослідницький звіт (1–2 сторінки): вступ (обрана галузь та питання), опис 3–5 прикладів ШІ, таблиця переваг та ризиків, власна позиція з обґрунтуванням (5–8 речень). Список використаних джерел (щонайменше 3).
